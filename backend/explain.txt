아래에서 설명할 Prisma 지원된다는게 크고(Prisma가 아주 좋은 친구니까)
우리가 지지율, 투표율 이런 데이터를 많이 사용하니까 통계가 많이 쓰임.
쉽게 말해서 통계 쿼리가 큰 요구사항인데, 이 때 GROUP BY나 집계함수를 야무지게 제공해주는 DB가 PostgreSQL임
그리고 투표가 있는 만큼 투표 데이터 정합성이 굉장히 중요한데, ACID 트랜잭션을 보장해서 좋음(포인트 차감이랑 투표 원자성)

MySQL도 나쁘진 않은데 그냥 PostgreSQL이 좀 더 좋아보였고, 더 많이 쓰여서 선택함

Redis
일단 캐시가 필요하긴 했음
캐시가 없으면 매 요청마다 DB 조회를 해야 되는데 너무 느림(요청이 굉장히 많을텐데.....)
일단 SSE 실시간 업데이트에서 빠른 pub/sub 된다는 점에서 좋고
JWT를 Access Token과 Refresh Token으로 나눠뒀는데 Refresh Token을 저장할 때 TTL 자동 만료 기능 때문에 좋음
그리고 아까 위에서 통계 중요하다고 했는데, 통계 캐싱 면에서의 장점이 확실함. 왜냐면 Redis는 초고속 읽기(메모리) 지원하니까.
